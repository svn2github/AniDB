UDP clients, WebAOM, and development

29.10.05 17:23 by rar- edit - del
A little publicised feature of AniDB is the UDP interface - a simple, lightweight way of talking directly to the database, useful for creating fun little applications. While not intented to replace the fully featured TCP interface used by AOM, it can already be used for specific tasks like notifications and mylist management.

The most complete usage so far is epoximator`s very handy WebAOM - a Java file hashing and mylist update program that should prove invaluable for anyone who`s been unable to use AOM due to its Windows dependance.

And for the actual recent news, development of the UDP interface has followed in the footsteps of the Chii code and been delegated away from the site`s busiest member. So, if you`re interested in writing a program, have a look at the current UDP API specs, and if you have any suggestions for improvement/additions, throw them at the API dev page, for the attention of the newly appointed maintainer, epoximator.

more special madness

22.08.05 00:31 by deridiot- edit - del
As of today "Specials" got broken into several parts. This makes it easier to sort out content and allows us to get rid of parody eps without the need of deleting them.

The following Specials are available:
  • Special/Extra Episode (DVD Specials, Bonus eps)
  • Opening/Ending/Theme/Credits Episode
  • Trailer/Promo/Adv Episode
  • Parody/Fandub Episode
  • Other Episode (Multi and Split-ep Files (movies and 1ep ovas with multiple parts listed under regular eps are excluded of this), Supportfiles (AC3 modpacks, fontpacks, subtitles for more than 1 ep) and other things.)

"Other Episode" is NOT meant for OST`s, Manga, AMV`s or any other not official Special!!!!

There is no change of the rule for allowed content.

You will find the option "Hide parody eps on anime page" in your profile. Once activated all eps with the special type "Parody/Fandub" will be hidden on the animepages.

The complete Changelog can as always be found here: here.

new AOM released

03.08.05 05:29 by deridiot- edit - del
PetriW released a new version of AOM (V0.5.5.238 (01.08.2005)) today.

The download link can like always be found on the AniDB Clients Wiki page

This release fixes the 100% CPU load bug that was encountered quite often recently while starting AOM.
Furthermore it allows to autocreq length and bitrate of files.

Some notes about the new autocreqs:
  • There is zero tolerance for length. Meaning there will be changes of 1sec if necessary.
  • For audio bitrate there exist a 10% tolerance and for video bitrate a 15% tolerance. This means if the local data (what AOM calculates on your pc for the file) differs less than the mentioned values to AniDB’s existing data it won’t issue a creq for them.

!IMPORTANT NOTICE!
This is not guaranteed to be correct (and what’s correct in the world of bitrates anyway?), there are already a few files marked to be ignored by AoM.
If you notice a file being creqed incorrectly you can add the text "!AOMSKIP!" to it’s description and AoM will ignore it. Once it’s been marked you don’t really need to notify me as I can find marked files with ease.
I’ll use the files marked !AOMSKIP! to trace down what’s wrong.
!IMPORTANT NOTICE!

Please notice there’ll be a new version with better codec support asap. With any luck, I`ll include MKV support.

Update on recent changes

03.07.05 05:03 by deridiot- edit - del
Various updates took place again today. Though most of them are mod only or don`t affect the average user directly. But let me name the one which is rather significant and might confuse some users at first:

file add dialog no longer loads the entire group list as a dropdown
What does that mean? Well before we had 3 lists in dropdown menus in the add/edit file dialog for the groups. This made some sense while we still had only a couple hundred groups in the DB. But today we have thousands of groups listed in our DB, meaning the lists were long, confusing and took time to load.
This was changed today. The first 2 lists have been replaced by a searchfield, which allows you to enter the group name. After clicking search, a dropdown menu with the results will be displayed.
This should make it easier for people to add and edit files.
Note: The 3rd list "by relation" was left in place, as it was the only useful one of the three.

Furthermore Mods now have the ability to merge placeholder files with generic files today. Meaning we can finally get rid of those evil things without affecting the mylists of users. This is a big task since those files are all over the DB, so if you notice such a file please report it here.

As always, the complete Changelog can be found here.

Also, two days ago "Categories" (the new genre system of AniDB) was opened to the average user. This means everyone can now add new categories to anime entries.
Some notes about it:
  • A wikipage which explains all categories, the rating and so on, is still in the works.
  • The categoriestree isn`t finished just yet. Some things might get removed, added or moved around. So this should be considered as a beta phase.
Ideas and all discussion about how to improve the overall system: here
Requests for Categories (Add/Rename/Move/Delete): here
